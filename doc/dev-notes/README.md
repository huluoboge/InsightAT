# 开发笔记（dev-notes）

本目录存放**开发过程中的记录与思考**，多为与 Agent 协作完成或实现某功能时写下的说明、修复报告、迁移总结等。

## 性质说明

- **过程稿，非最终规范**：内容反映当时的实现思路或决策，最终代码与设计可能已调整，**以 `doc/design/` 与仓库代码为准**。
- **供参考与追溯**：便于回顾某次修复/重构的来龙去脉，或理解某模块的实现背景；新读者若与当前行为不一致，可优先查阅 `design/` 与源码。
- **可随时增删**：新写的过程记录可放在本目录或子目录；若某篇已过时，可归档或删除，无需视为正式文档维护。

## 目录结构

```
dev-notes/
├── README.md                    ← 本说明
├── *.md                         ← 各类实现报告、修复总结、进度记录
│   ├── CODE_ANALYSIS_ROTATION.md   ← 旋转代码问题分析（过程稿）
│   └── ROTATION_PROJECT_SUMMARY.md ← 旋转项目过程总结（过程稿）
└── third_party/                 ← 第三方库迁移、集成过程记录
    ├── siftgpu/
    └── superpoint/
```

## 与 design/ 的区别

| 类型     | 位置        | 用途                         |
|----------|-------------|------------------------------|
| 设计文档 | `doc/design/` | 架构、数据模型、规范等**权威参考** |
| 开发笔记 | `doc/dev-notes/` | **实现过程**记录，思路与历史参考   |
