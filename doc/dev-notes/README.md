# 开发笔记（dev-notes）

本目录存放**开发过程中的记录与思考**，多为与 Agent 协作或实现某功能时写下的说明、修复报告、迁移总结等。

## 性质说明

- **过程稿，非最终规范**：内容反映当时的实现思路或决策，可能与当前代码不一致，**以仓库代码为准**；设计类长文见本目录下的 **`design/`**。
- **供参考与追溯**：便于回顾某次修复/重构的来龙去脉。
- **可随时增删**：过时文档可归档或删除，无需视为对外承诺。

## 目录结构

```
dev-notes/
├── README.md              ← 本说明
├── build.md               ← 构建、依赖、Docker 等（维护时与根 README 对齐）
├── design/                ← 历史设计文档（原 doc/design/）
│   ├── index.md
│   ├── rotation/
│   └── retrieval/
├── *.md                   ← 各类实现报告、修复总结、进度记录
└── third_party/           ← 第三方库迁移、集成过程记录
    ├── siftgpu/
    └── superpoint/
```

## 与「正式对外文档」的区别

| 类型 | 位置 | 用途 |
|------|------|------|
| 对外概述 / 许可 | 仓库根目录 `README.md`、`LICENSE`、`THIRD_PARTY_LICENSES.md` | 用户与合规入口 |
| 历史设计稿 | `doc/dev-notes/design/` | 架构、约定、算法思路（可能滞后） |
| 开发笔记 | `doc/dev-notes/*.md` 等 | 实现过程、排障、计划草案 |
