## Sparse reconstruction pipeline

- **One-shot driver** — `isat_sfm` runs the usual stages: project / features / retrieval or exhaustive pairs / match / geometry / tracks / incremental SfM, with `--steps` to run a subset or resume.
- **GPU feature front end** — SIFT feature extraction and matching on GPU. Default build uses **PopSift** (CUDA). You can also select **SiftGPU** (often faster) with **`isat_sfm --use-sift-gpu`** (and matching/extract backends as documented in `--help`). SiftGPU is under a **non-commercial** upstream license (educational, research, and non-profit use); see [THIRD_PARTY_LICENSES.md](../../../THIRD_PARTY_LICENSES.md) and `third_party/SiftGPU/`.
- **SiftGPU: CUDA vs GLSL (EGL)** — The **CUDA** SiftGPU path uses NVIDIA’s CUDA only; it does **not** use EGL or X11. Upstream SiftGPU in this project is intended for **CUDA 11.8**; **use CUDA 11.8** when you rely on the SiftGPU CUDA backend. The **OpenGL/GLSL (shader) SiftGPU** path can drive the GPU without CUDA; for **headless** runs without a full X11 desktop, it can use an **EGL** offscreen context (see [doc/develop/build.md](../build.md) and [DOCKER_BUILD.md](../../../DOCKER_BUILD.md) for extract/match backend flags). Use the GLSL path when **no CUDA** is available or you explicitly choose that backend; CUDA behaviour is separate and **unchanged** by EGL.
- **Outputs (incremental SfM)** — the final `incremental_sfm` step writes, by default:
  - **Bundler-style** `bundle.out` and `list.txt` under the work directory (e.g. `…/incremental_sfm/`) for viewers and external tools;
  - **COLMAP text sparse** under **`…/incremental_sfm/colmap/sparse/0/`** (`cameras.txt`, `images.txt`, `points3D.txt`) for dense or COLMAP-based workflows.
- **Watch reconstruction grow** — use **`isat_sfm --output-interval-sfm`**. That creates **`<work-dir>/sfm_interval/`** and writes a **Bundler snapshot each SfM iteration** under `sfm_interval/iter_NNNN/bundle.out` (and `list.txt` beside it), with a fixed **interval of 1** (no extra knobs). Open those paths in **`at_bundler_viewer`** to scrub through the reconstruction. (`isat_incremental_sfm` still has low-level `--debug-dir` / `--debug-interval` / `--bundler-max-cameras` if you need manual control; the pipeline driver keeps the friendly switch above only.)

- **HTML quality report** — the repo includes **`scripts/isat_report.py`**, a **stdlib-only** script that ingests a completed **work directory** and writes a **single self-contained dark-theme HTML** report (summary, keypoint and pair stats, inlier histograms, tracks, reprojection and per-image tables). Example:

  ```text
  python3 scripts/isat_report.py --work-dir /path/to/work --output report.html
  ```

  If your SfM output lives only under `incremental_sfm/`, you can point `--sfm-dir` at that path; see the script’s docstring. When using an **AppImage**, the script is under the image’s `usr/share/InsightAT/scripts/` and `python3` may be bundled there—see [packaging/appimage/README.md](../../../packaging/appimage/README.md).
